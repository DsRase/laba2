# Лаба 2: классический WAN на IPsec

Сначала выдаём скрипту права и прогоняем его: `chmod +x forge_tunnel.sh && ./forge_tunnel.sh`

Дальше поднимаем стенд: `docker compose up --build -d`

Проверим, что туннель живой и пинг между филиалами идёт: `docker exec branch_a ping -c 3 192.168.77.2`

Теперь смотрим, какой интерфейс смотрит в основной канал (у этой линии ip 10.50.0.11): `docker exec -it branch_a ip -br addr`

Когда нашли интерфейс — глушим его через netem (подставьте свой `<iface>`): `docker exec -it branch_a tc qdisc change dev <iface> root netem loss 100%`

Снова пробуем пинг: `docker exec branch_a ping -c 3 192.168.77.2`. Связи нет — это и есть слабое место статического IPsec: SA жёстко привязана к одному транспортному адресу, резервный канал (10.60.0.0/24) физически на месте, но конфиг про него не знает и ничего автоматически не переключит.
